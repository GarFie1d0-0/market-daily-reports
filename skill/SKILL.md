# 全球金融市场深度日报 (market-daily)

## Overview

当用户请求生成全球金融市场报告时，按照本 Skill 定义的 7-Phase 流程执行：

```
Phase 1: 并行数据采集（15-20 搜索）    → 素材收集
Phase 2: 多 Agent 并行深度分析 ★核心   → 5 角色 AI 投资委员会并行分析
Phase 3: 主 Agent CIO 整合    ★新增    → 通读 5 份报告 + 标记矛盾
Phase 4: 质量检查清单                    → 数据完整性 + 指标覆盖审查
Phase 5: 多源交叉验证与分歧展示          → 增强：优先引用风控官发现
Phase 6: 报告编译                       → 来源标注为"各 Agent + CIO"
Phase 7: 双路径存档与版本控制            → 主路径 + Git 备份
```

**v2.0 核心升级**: Phase 2 引入 5 角色 AI 投资委员会架构——用角色分工制造"认知多样性"，用独立风控官角色防止群体思维。这跟多 agent 分析模式本质一致：不同角色从不同视角审视同一组数据，风控官专门找矛盾、堵漏洞、推演尾部风险。

**触发短语**: "市场日报" / "全球市场报告" / "market daily" / "今日行情" / "financial market report" / "市场分析报告"

### Phase 2: 多 Agent 并行深度分析（★ v2.0 核心升级）

这是 v2.0 最关键的阶段。**一次性并行启动 5 个 SubAgent**，每个扮演不同的投资委员会角色，从不同视角独立分析同一组原始搜索数据。

#### 架构原理

用角色分工制造"认知多样性"（cognitive diversity），用独立风控官角色防止群体思维（groupthink）。5 个角色各写各的报告，互不知晓对方结论——这确保每份分析是独立判断而非相互附和。

```
┌─────────────────────────────────────────────────────┐
│              AI 投资委员会 (Phase 2)                  │
├───────────────┬───────────────┬───────────────────────┤
│ 宏观策略师     │ A股深度分析师   │ 海外市场分析师         │
│ (全球宏观)     │ (A股技术+情绪)  │ (港股+美股+日股)       │
├───────────────┼───────────────┼───────────────────────┤
│ 加密分析师     │ 风控与交叉验证官 │                      │
│ (BTC/ETH+链上) │ (审查+找矛盾)   │                      │
└───────────────┴───────────────┴───────────────────────┘
```

| 角色 | 职责范围 | 制衡关系 |
|------|---------|:--:|
| 宏观策略师 | 全球流动性、地缘政治、央行政策、跨市场共振、风险偏好 | 被风控官审查 |
| A 股深度分析师 | 技术面 + 市场情绪 + 板块轮动 + 龙虎榜 + 机会识别 + 风险预警 | 被风控官审查 |
| 海外市场分析师 | 港股 + 美股 + 日股独立分析（技术面 + 资金流 + 事件） | 被风控官审查 |
| 加密分析师 | BTC/ETH 技术面 + 链上数据 + 鲸鱼行为 + 监管动态 | 被风控官审查 |
| **风控与交叉验证官** | 审查其他 4 人报告、找矛盾与逻辑漏洞、推演极端尾部情景 | **制衡所有角色** |

#### SubAgent 5: 风控与交叉验证官（★ 关键制衡角色）

**角色定位**: 独立风控官（Risk Officer），不负责产出原创分析，而是**审查前 4 份分析报告的质量和一致性**。这是防止群体思维（groupthink）和过度乐观的最后防线。

**⚠️ 关键约束**: 这个 SubAgent 必须在其他 4 个 SubAgent **全部完成后再启动**。它需要阅读所有 4 份报告来进行交叉验证。

**硬性约束**:
1. **必须至少找出 3 条矛盾**——可以是不同分析师对同一数据的解读差异、逻辑链断裂、或数据与结论不匹配
2. **必须检测回声室效应**——如果多个分析师用相似的论据得到相似结论，必须警告可能存在信息茧房
3. **必须推演 1-2 个极端尾部情景**——假设最坏情况发生，传导链条是什么？
4. **可以不同意任何分析师**，但必须给出具体理由和反证

**输出要求**（写入临时文件 `workspace-files/.market-daily/agent-5-risk.md`）:

### Phase 3: 主 Agent CIO 整合（★ v2.0 新增）

你是 CIO（首席投资官），**通读全部 5 份 SubAgent 报告**，完成整合：

**Step 1: 逐份审阅**
- 通读 5 份报告，理解每个角色的核心判断
- 标记各报告中的关键结论和核心论据

**Step 2: 矛盾地图**
- 重点关注风控官（Agent 5）找出的矛盾
- 判断哪些矛盾是"真分歧"（基于不同前提假设）、哪些是"视角差异"（因覆盖范围不同）
- 对于真分歧，追溯到根本的前提假设差异

**Step 3: 一致性判断**
- 4 位分析师在哪些判断上高度一致？（标记为"强共识"）
- 哪些判断存在明显分歧？（标记为"待观察"）
- 风控官的回声室警告是否成立？

**Step 4: 形成整合叙事**
- 综合所有角色观点，形成统一的跨市场叙事
- 明确标注哪些判断来自哪个 Agent（如"[宏观策略师] 认为流动性正在收紧"）
- 有分歧的地方如实呈现，不强行统一

### Phase 4: 质量检查清单

在进入报告编译前，逐项核查：

```
□ 每个市场是否覆盖了完整技术指标（支撑/阻力/均线/RSI/MACD/成交量）？
□ 数据是否存在明显矛盾或异常？（如有，回溯重新确认）
□ 所有数值是否有来源依据？（没有编造数据）
□ 风控官找出的矛盾是否都得到了处理？
□ 是否有重要市场或指标被遗漏？
□ 极端尾部情景是否已经纳入风险矩阵？
```

如果任何一项不通过，回到对应 Phase 补充后重检。

### Phase 5: 多源交叉验证与分歧展示（增强版）

**v2.0 增强**: 优先引用风控官（Agent 5）在 Phase 2 中已经发现的矛盾和逻辑漏洞。

## 关键原则

### 1. 认知多样性 > 单一深度
v2.0 的核心价值在于 5 个角色从不同视角独立分析同一组数据。不要追求单个角色的"绝对深度"，而是追求角色间的"认知差异"——不同角色应该用不同的分析框架和关注重点。

### 2. 制衡机制 > 共识输出
风控官角色是本架构的关键。她必须能够推翻或质疑任何分析师。如果风控官没有找出任何问题，要么是分析过于浅显，要么是风控官没有尽责。

### 3. 数据完整性 > 速度
宁可多花一轮搜索，也要确保每个市场的数据完整。如果某些指标（如 RSI、MACD）搜索不到具体数值，根据价格行为做定性推断并标注"基于价格行为推断"。

### 4. 分歧展示 > 观点输出
本报告的价值在于**展示市场正在交易什么逻辑**，而非给出确定性的多空结论。当多空双方都有合理逻辑链时，完整展示双方论证，帮助用户自己判断。

### 5. 逻辑链 > 数据罗列
每个技术指标和事件都要回答"所以呢？"——它通过什么机制影响市场？传导链条是什么？谁会受益、谁会受损？

### 6. 中文为主，保留英文术语
报告主体用中文撰写，但保留英文金融术语（RSI、MACD、MVRV、SOPR、FOMC 等），不做生硬翻译。

### 7. 不要制造假数据
如果搜索结果中某个指标确实没有具体数值，**不要编造**。标注"未获取到具体数值"，用可获取的相关信息做定性描述。

## 常见问题处理

**Q: 搜索结果质量差/返回了去年的数据？**
A: 调整搜索词，加入更精确的日期。如果仍无改善，在报告中标注"部分数据基于最近可获取信息（日期）"。

**Q: 某个市场当天休市？**
A: 标注"XX 市场今日休市"，使用前一个交易日数据并注明。

**Q: 用户只关注特定市场？**
A: 仍执行完整流程，但用户指定的市场做更深入分析，其他市场保持标准深度。

**Q: 报告太长，用户说太啰嗦？**
A: 保留一分钟速览 + 执行摘要 + 各市场行情快照 + 多空分歧 + AI 投资委员会观点全景，可以缩减技术分析和事件部分，但**多空分歧展示和风控官警告不可省略**。

**Q: SubAgent 返回的报告质量差/格式不对？**
A: 检查 SubAgent 的 prompt 是否包含了完整的输出格式要求。如果某个 Agent 返回质量不佳，用更具体的指令重新启动该 Agent。

**Q: 风控官没有找到 3 条矛盾？**
A: 这是严重问题。重新启动风控官 SubAgent，强调其硬性约束：必须找出至少 3 条矛盾。如果分析确实高度一致，要求风控官推演"如果市场共识错了会怎样"。
