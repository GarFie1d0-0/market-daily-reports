# 全球金融市场深度日报 (market-daily)

## Overview

当用户请求生成全球金融市场报告时，按照本 Skill 定义的完整流程执行：并行搜索 → 技术分析 → 多源交叉验证 → 编译报告 → Git 存档。

**核心价值**: 不只是数据罗列，而是提供技术面分析、多空逻辑链对比、AI 源分歧展示，帮助用户理解市场"正在交易什么逻辑"而非仅仅"发生了什么"。

**触发短语**: "市场日报" / "全球市场报告" / "market daily" / "今日行情" / "financial market report" / "市场分析报告"

**报告存档路径**: `workspace-files/reports/YYYY-MM-DD.md`（Git 版本控制）

### Phase 2: 技术分析深度提炼

对每个市场，从搜索结果中提取并组织以下内容：

**必须覆盖**:
```
□ 当前点位 / 涨跌幅
□ 支撑位（至少 3 个递进层级）
□ 阻力位（至少 3 个递进层级）
□ 均线状态（多头/空头排列，价格与 MA 的关系）
□ RSI（数值 + 超买/超卖判断）
□ MACD（金叉/死叉状态 + 动能方向）
□ 成交量/成交额（与前一交易日对比 + 趋势）
□ 关键技术形态（头肩顶/底、三角形、通道等）
□ 资金流向（北向/南向/机构/散户）
```

**A 股专属**: 融资余额、北向资金、涨跌比、涨停家数
**美股专属**: 市场广度（站上200日均线的比例）、Mag7 表现分化
**日股专属**: 腾落比率、空売り比率、PER、SQ 日期
**加密专属**: MVRV、Realized Price、交易所净流量、稳定币供给、鲸鱼地址行为

### Phase 3: 多源交叉验证与分歧展示（★ 核心差异化价值）

这是本报告最有价值的部分。**不要简单汇总信息，要找出矛盾**。

**操作步骤**:
1. 对比不同来源对同一指标的数据（如点位、估值、资金流向）
2. 对比多头和空头对同一数据的**不同解读**
3. 当出现分歧时，完整展示两方逻辑链，不做武断结论
4. 追溯分歧的**根源假设**——双方在哪个前提上产生了差异？

**分歧展示模板**:

```markdown
### 🔴🟢 多空分歧

#### 多头逻辑链（[来源机构/分析师]）
> 核心论点: [一句话]

逻辑链:
  前提A → 推论B → 结论C

#### 空头逻辑链（[来源机构/分析师]）
> 核心论点: [一句话]

逻辑链:
  前提X → 推论Y → 结论Z

#### ⚡ 分歧逻辑链解析

| 数据点 | 多头解读 | 空头解读 |
|--------|----------|----------|
| [同一数据] | [乐观解释] | [悲观解释] |

**分歧根源**: [双方在哪个前提假设上产生了根本差异]

**验证方式**: [后续可以通过观察什么指标来判断哪方正确]
```

## 关键原则

### 1. 数据完整性 > 速度
宁可多花一轮搜索，也要确保每个市场的数据完整。如果某些指标（如 RSI、MACD）搜索不到具体数值，根据价格行为做定性推断并标注"基于价格行为推断"。

### 2. 分歧展示 > 观点输出
本报告的价值在于**展示市场正在交易什么逻辑**，而非给出确定性的多空结论。当多空双方都有合理逻辑链时，完整展示双方论证，帮助用户自己判断。

### 3. 逻辑链 > 数据罗列
每个技术指标和事件都要回答"所以呢？"——它通过什么机制影响市场？传导链条是什么？谁会受益、谁会受损？

### 4. 中文为主，保留英文术语
报告主体用中文撰写，但保留英文金融术语（RSI、MACD、MVRV、SOPR、FOMC 等），不做生硬翻译。

### 5. 不要制造假数据
如果搜索结果中某个指标确实没有具体数值，**不要编造**。标注"未获取到具体数值"，用可获取的相关信息做定性描述。

## 常见问题处理

**Q: 搜索结果质量差/返回了去年的数据？**
A: 调整搜索词，加入更精确的日期。如果仍无改善，在报告中标注"部分数据基于最近可获取信息（日期）"。

**Q: 某个市场当天休市？**
A: 标注"XX 市场今日休市"，使用前一个交易日数据并注明。

**Q: 用户只关注特定市场？**
A: 仍执行完整流程，但用户指定的市场做更深入分析，其他市场保持标准深度。

**Q: 报告太长，用户说太啰嗦？**
A: 保留一分钟速览 + 执行摘要 + 各市场行情快照 + 多空分歧，可以缩减技术分析和事件部分，但**多空分歧展示不可省略**。
